# SendToERP — макет кнопок для Outlook на веб

## Важное ограничение Outlook на веб

В классическом Outlook для Windows сторонние (COM/VSTO) надстройки могут добавлять
**отдельную вкладку** на ленту главного окна (как "SPS" на скриншоте). Это технология,
доступная только в Windows-десктопном Outlook.

**Outlook на веб (а также новый Outlook для Windows и Outlook для Mac) такой возможности
не дают.** Максимум, что позволяет современная модель надстроек (Office Add-ins) — это
добавить **свою группу кнопок на существующую вкладку** ленты: "Главная" — когда письмо
открыто для чтения, и "Сообщение" — когда письмо создаётся/редактируется.

Поэтому ниже воссоздана не отдельная вкладка, а группа **"SendToERP (Smadar)"** с теми
же 4 кнопками, размещённая на вкладке "Главная" — это максимально близкий аналог,
который реально работает в Outlook на веб.

## Что внутри

- `manifest.xml` — манифест надстройки (формат Add-in Commands, понимается Outlook на веб)
- `commands.html` / `commands.js` — код, который выполняется по нажатию кнопок
- `assets/*.png` — иконки кнопок (16/32/80 px), стилистически похожи на оригинал
- `fallback.html` — страница-заглушка для очень старых клиентов

## Текущее состояние

Это пока **только макет интерфейса**, как вы и попросили: кнопки реально появляются на
ленте и реагируют на клик (показывают всплывающее уведомление в письме), но никакой
реальной отправки в ERP не происходит. Логику отправки можно добавить позже в
`commands.js`, когда будет готов API ERP-системы.

## Как это запустить

### 1. Разместите файлы на HTTPS-адресе

Outlook на веб загружает все файлы надстройки по реальным HTTPS-ссылкам — локальные
файлы он не видит. Варианты:

- быстро и бесплатно: GitHub Pages (загрузить эту папку в репозиторий, включить Pages);
- внутренний веб-сервер компании;
- для разовой проверки — локальный dev-сервер по `https://localhost:3000` (его видит
  браузер, даже когда сама страница Outlook открыта на office.com).

### 2. Замените адрес в манифесте

В файле `manifest.xml` есть несколько строк с `https://YOUR-DOMAIN` — замените их на
реальный адрес, где лежат файлы (например `https://username.github.io/sendtoerp`),
сохранив остальную часть пути (`/assets/...`, `/commands.html` и т.д.).

### 3. Установите надстройку в Outlook на веб

В Outlook на веб: откройте любое письмо → значок "..." (Дополнительные действия) или
шестерёнка настроек → **Управление надстройками** / **Получить надстройки** → **Мои
надстройки** → **"+" Добавить личную надстройку** → **Добавить из файла…** → выберите
отредактированный `manifest.xml`.

### 4. Проверьте результат

Откройте любое письмо для чтения — на вкладке **Главная** должна появиться группа
**SendToERP (Smadar)** с четырьмя кнопками. То же самое — при создании нового письма,
на вкладке **Сообщение**.
